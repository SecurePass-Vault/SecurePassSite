Security Architecture

Dual Encryption Layer

Your vault is protected with a dual-key system: your master password and a recovery key. Either can decrypt your vault independently, ensuring you never lose access to your passwords.

PBKDF2 Key Derivation

Uses PBKDF2 with SHA-256 and 100,000 iterations to derive encryption keys from your password, making brute-force attacks computationally infeasible.

AES-256-GCM Encryption

All vault data is encrypted using AES-256-GCM, which provides both confidentiality (AES-256) and authenticity (Galois/Counter Mode) to detect tampering.

Cryptographically Secure Random Generation

All salts, nonces, and keys are generated using cryptographically secure random number generators to ensure unpredictability.

Session Management

Automatic session locking, secure in-memory handling, and proper cleanup of sensitive data prevents unauthorized access.

Vault File Format

Vault files contain an encrypted header with key material and a fully encrypted payload, making it impossible to extract information without the master password or recovery key.
